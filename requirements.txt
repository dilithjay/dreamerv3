# DreamerV3 (JAX) dependencies, adapted for the Digital Research Alliance of
# Canada (Compute Canada) wheelhouse. See SLURMs/install_venv.sh for how these
# map onto `pip install --no-index` (wheelhouse) vs PyPI `--no-deps` fallbacks.
#
# Key deviations from upstream dreamerv3 for the Alliance environment:
#   * jax pinned to 0.5.0 (matches the version this dreamerv3 is tested against in
#     the Dockerfile AND is available in the Alliance wheelhouse). Installed with
#     --no-index so the matching jaxlib + jax-cuda12 plugin/pjrt come from the
#     wheelhouse under the loaded cuda/12.2 module. Do NOT use the [cuda12] extra
#     here (that pulls nvidia-* CUDA wheels from PyPI) unless using the PyPI
#     fallback documented in install_venv.sh.
#   * nvidia-cuda-nvcc-cu12 dropped (only needed for the PyPI-CUDA fallback path).
#   * dm_control 1.0.28 (wheelhouse) + mujoco 3.3.0 (PyPI prebuilt wheel; the
#     Alliance wheelhouse does NOT carry mujoco). Modern stack, python 3.11. The
#     legacy py3.8 dm_control 0.0.x that registered a `reacherloca` suite domain
#     is incompatible with python 3.11 / jax 0.5.0, so the reacherloca task is
#     vendored (embodied/envs/reacherloca_task.py + reacherloca.xml) and built
#     directly -- no suite-domain registration needed. The mujoco wheel is
#     self-contained (bundles libmujoco + EGL/GLFW/OSMesa); no native staging.
#   * gym-minigrid + classic gym added for the MiniGrid LoCA env.
